# Roofline Model Analysis: CUDA Median Filter (v8)

Questo documento analizza le prestazioni del kernel v8_vertical_sliding.cu utilizzando il modello Roofline.

## 1. La Teoria: Cos'è il Roofline Model?
Il modello mette in relazione la Prestazione (GFLOPS/GIOPS) con l'Intensità Aritmetica (AI) del programma.
Il "soffitto" (roof) è composto da due rette:
1. Memory Bound: La parte inclinata, dove la performance è limitata dalla banda passante della memoria (GB/s).
2. Compute Bound: La parte orizzontale, dove la performance è limitata dalla potenza di calcolo della GPU (GIOPS).

Il punto di incontro tra le due si chiama Ridge Point.

## 2. Dati Hardware (NVIDIA RTX 3050 Laptop)
* Architettura: Ampere (SM 8.6)
* CUDA Cores: 2048
* Clock (Lockato): 1500 MHz
* Peak Compute (INT32): 2048 cores * 1.5 GHz * 1 op/ciclo = 3072 GIOPS
* Peak Bandwidth: 128 GB/s (valore tipico per bus a 128-bit GDDR6)
* Ridge Point: 3072 / 128 = 24 ops/byte

## 3. Analisi del Programma (v8_vertical_sliding)
Per ogni pixel elaborato (3 canali R, G, B):

### A. Carico di Lavoro (Operazioni)
L'algoritmo di mediana veloce (fast_median_scalar) esegue circa 36 operazioni intere (min, max) per canale.
* Ops Totali per Pixel: 36 * 3 canali = 108 ops/pixel

### B. Traffico Dati (Memoria)
Grazie al Vertical Sliding Window, ogni pixel viene letto dalla memoria globale idealmente una sola volta (per canale) e scritto una volta.
* Byte Totali per Pixel: 3 canali (Read) + 3 canali (Write) = 6 bytes/pixel

### C. Intensità Aritmetica (AI)
AI = Operazioni / Byte = 108 / 6 = 18 ops/byte

## 4. Conclusioni del Modello
Poiché la tua AI (18) è inferiore al Ridge Point (24), il tuo programma è MEMORY BOUND.

* Performance Massima Teorica: AI * Bandwidth = 18 * 128 = 2304 GIOPS
* Tempo Minimo Teorico (SOL):
  Per un'immagine da 12288 x 16320 (200.5 Mpix):
  T_min = (200.5 Mpix * 108 ops) / 2304 GIOPS = 9.4 ms

Verifica Empirica: Il tuo tempo di 12 ms indica un'efficienza del 78% rispetto al limite fisico. È un risultato eccezionale!
